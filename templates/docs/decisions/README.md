# 架构决策记录 (ADR)

本目录存储所有架构决策记录 (Architecture Decision Records)。

## 什么是 ADR？

ADR 是记录重要架构决策的轻量级文档，每个决策一个文件。它回答：
- **做了什么决策**
- **为什么做这个决策**
- **考虑过哪些替代方案**
- **有什么后果**

## 命名规范

```
ADR-{编号}-{kebab-case-title}.md
```

示例：
- `ADR-001-database-choice.md`
- `ADR-002-api-design.md`
- `ADR-003-auth-strategy.md`

## 状态定义

| 状态 | 说明 |
|------|------|
| Proposed | 提议中 |
| Accepted | 已接受（不可修改） |
| Deprecated | 已弃用 |
| Superseded by ADR-XXX | 被新 ADR 替代 |

## ADR vs 提案

| 文档类型 | 用途 | 粒度 |
|---------|------|------|
| 提案 | 完整功能需求 | 功能级别 |
| ADR | 单一架构决策 | 决策级别 |

一个提案可能产生多个 ADR。

## 最佳实践

1. **一页纸原则**：ADR 应该简洁，聚焦于决策本身
2. **不可修改**：接受后的 ADR 不修改，通过新 ADR 来替代
3. **双向链接**：如果 ADR-005 替代 ADR-003，两者都要互相引用
4. **指定 Owner**：每个 ADR 有明确的负责人
