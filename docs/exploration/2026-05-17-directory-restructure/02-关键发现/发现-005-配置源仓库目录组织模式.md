# 发现-005：配置源仓库目录组织模式

**发现时间**：2026-05-17
**发现来源**：librarian agent（ai-config、cc-settings、GNU Stow 分析）

---

## 核心发现

### 1. ai-config 的目录模式

```
ai-config/                    # 源仓库（清晰目录）
├── agents/                   # 不使用隐藏目录
├── commands/
├── skills/
├── hooks/
├── instructions/
│   └── global.md
├── settings/
│   └── mcp.ts
├── scripts/
│   ├── adapters/            # 平台适配器
│   │   ├── claude-code.ts
│   │   └── opencode.ts
│   └── index.ts             # 安装器主逻辑
└── bin/                      # npx 入口

# 安装后
~/.claude/                    # 全局模式（隐藏目录）
项目/.claude/                 # 项目模式（隐藏目录）
```

**关键点**：
- 源仓库使用**清晰目录名**（`commands/`、`agents/`）
- 安装时才创建**隐藏目录**（`.claude/`、`.opencode/`）
- 支持两种安装范围：**Project**（项目级）和 **Home**（用户级）

### 2. cc-settings 的目录模式

```
cc-settings/                  # 源仓库
├── agents/
├── skills/
├── rules/
├── hooks/
├── config/                   # 分层配置片段
│   ├── 10-core.json
│   ├── 20-mcp.json
│   └── 30-permissions.json
├── src/
│   └── setup.ts              # 安装器
├── CLAUDE-FULL.md            # 安装为 ~/.claude/CLAUDE.md
├── CLAUDE.md                 # 精简版
└── AGENTS.md                 # 跨工具标准
```

**关键创新**：
- `config/*.json` 按数字前缀分层组合
- `CLAUDE-FULL.md` → 安装为 `CLAUDE.md`
- 备份机制 + 版本追踪

---

### 3. GNU Stow 的目录模式

```
dotfiles/                     # 源仓库
├── nvim/                     # 包名
│   └── .config/              # 镜像目标路径
│       └── nvim/
│           └── init.lua      # → ~/.config/nvim/init.lua
├── zsh/
│   └── .zshrc                # → ~/.zshrc
└── install.sh                # stow -R -t $HOME */
```

**关键原理**：
- 源目录结构**镜像目标路径**
- 安装命令：`stow <package>`
- 目标路径：自动映射到 `$HOME`

## 对 tg-workflow 的启发

### 源目录设计原则

1. **使用清晰目录名**：`configs/claude/` 而非 `.claude/`
2. **按平台分组**：`claude/`、`opencode/` 子目录
3. **安装器独立**：`scripts/` 或 `src/` 目录

### 目标目录映射

```
tg-workflow/                  # 源仓库
├── configs/
│   ├── claude/              # 清晰目录
│   │   ├── commands/
│   │   ├── skills/
│   │   ├── hooks/
│   │   └── settings.json
│   └── opencode/
│       ├── commands/
│       ├── skills/
│       └── plugins/
└── scripts/
    └── install.sh

# 安装后
项目/.claude/                 # 隐藏目录
项目/.opencode/               # 隐藏目录
```

### 安装脚本模式

```bash
# 项目模式：复制到项目
cp -r configs/claude/* 项目/.claude/
cp -r configs/opencode/* 项目/.opencode/

# 全局模式：复制到用户目录
cp -r configs/claude/* ~/.claude/
cp -r configs/opencode/* ~/.config/opencode/

# 自用模式（tg-workflow 自身）：软链接
ln -s configs/claude .claude
ln -s configs/opencode .opencode
```
