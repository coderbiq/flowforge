# 发现-001：当前目录结构现状

**发现时间**：2026-05-17
**发现来源**：explore agent + glob/grep 直接搜索

---

## 核心发现

### 1. Commands 100% 重复

```
.claude/commands/tg/     → 7 个命令文件
.opencode/commands/tg/   → 7 个命令文件（内容完全相同！）
```

**验证**：`diff` 命令显示文件内容完全一致。

### 2. 目录结构分散

```
tg-workflow/
├── .claude/commands/tg/     # Claude Code 专用
├── .opencode/commands/tg/   # OpenCode 专用
├── hooks/                    # 独立目录（不属于任何工具目录）
├── plugins/                  # 独立目录
└── skills/                   # 共享资源
```

**问题**：hooks/plugins 不在任何工具特定目录，结构不一致。

### 3. 主项目现状

```
tangram-v2/.claude/
├── commands/opsx/          # 主项目自己的命令
├── skills/openspec-*/      # 主项目自己的 skills
└── settings.json           # hooks 配置

tangram-v2/.opencode/       # 不存在
```

**发现**：
- 主项目有自己的配置，命名空间不同（opsx vs tg）
- 无 `.opencode/` 目录
- **没有使用软链接** - 配置完全独立

---

## 冗余分析

| 配置类型 | 冗余程度 | 说明 |
|---------|---------|------|
| Commands | **100%** | 两个目录内容完全相同 |
| Hooks | 无重复 | 仅 Claude Code 使用 |
| Plugins | 无重复 | 仅 OpenCode 使用 |
| Skills | 无重复 | 被两个工具共同引用 |

---

## 安装复杂度

当前要安装 tg-workflow 到主项目，需要：
1. 复制 `.claude/commands/tg/` 到主项目
2. 复制 `.opencode/commands/tg/` 到主项目
3. 处理 `settings.json` 的 hooks 合并
4. 安装 hooks/plugins 到正确位置

**问题**：步骤多，容易出错，维护困难。

---

## 启发

1. **Commands 可以统一**：两个工具的命令格式完全兼容
2. **软链接未使用**：当前是独立维护，可以引入软链接简化
3. **Skills 已共享**：两个工具都引用相同的 Skill 定义
