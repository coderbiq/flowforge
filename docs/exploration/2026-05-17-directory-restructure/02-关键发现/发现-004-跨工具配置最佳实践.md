# 发现-004：跨工具配置最佳实践

**发现时间**：2026-05-17
**发现来源**：librarian agent（多工具对比 + 开源案例）

---

## 核心发现

### 1. AGENTS.md 是事实标准

**数据**：60,000+ 仓库采用 AGENTS.md 作为跨工具配置文件。

**支持工具**：
- Claude Code（作为 CLAUDE.md 替代）
- OpenCode（原生支持）
- Cursor（通过 `.cursor/rules/` 引用）
- GitHub Copilot（作为 `.github/copilot-instructions.md`）
- OpenAI Codex（原生支持）

### 2. 符号链接方案

**最佳实践**：
```bash
# AGENTS.md 作为单一真相源
ln -s AGENTS.md CLAUDE.md
ln -s AGENTS.md GEMINI.md
ln -s AGENTS.md .github/copilot-instructions.md
```

**注意事项**：
- Windows 需启用：`git config --global core.symlinks true`
- CI 需验证：`test -L CLAUDE.md`

### 3. 案例：azat-io/ai-config

**架构**：单一安装器，多代理支持

```
instructions/global.md    # 统一指令源
commands/                 # 统一命令
agents/                   # 统一代理
skills/                   # 统一技能
```

安装后分发到各工具目录。

### 4. 案例：darkroomengineering/cc-settings

**架构**：分层配置，渐进复杂度

| 层级 | 文件 | 用途 |
|------|------|------|
| Standards | AGENTS.md | 跨工具共享 |
| Claude Config | CLAUDE.md | Claude 特定 |
| Rules | rules/*.md | 按需加载 |

**效果**：核心配置减少 60% token 消耗。

### 5. 分层配置架构

```
┌─────────────────────────────────────┐
│   Team/Enterprise (最高优先级)       │
├─────────────────────────────────────┤
│   Project (版本控制,团队共享)        │
├─────────────────────────────────────┤
│   User (个人全局偏好)                │
├─────────────────────────────────────┤
│   Local (个人项目覆盖,不提交)         │
└─────────────────────────────────────┘
```

## 推荐设计模式

### 模式 1：单一真相源 + 符号链接

```
AGENTS.md              # 核心
├── CLAUDE.md → AGENTS.md
├── GEMINI.md → AGENTS.md
└── .github/copilot-instructions.md → AGENTS.md
```

### 模式 2：模块化组件

```
config-components/
├── core/               # 跨工具核心
├── tools/              # 工具特定
│   ├── claude/
│   └── cursor/
└── shared/             # 共享能力
    ├── agents/
    └── skills/
```

### 模式 3：Plugin 化分发

```bash
# 安装
claude plugin install github:org/tg-workflow
opencode plugin install github:org/tg-workflow
```

## 对 tg-workflow 的启发
