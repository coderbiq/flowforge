## 结论-001：Commands 统一方案

### 发现

1. `.claude/commands/tg/` 和 `.opencode/commands/tg/` 内容 **100% 相同**
2. OpenCode **自动加载** `.claude/skills/` 目录
3. 两个工具的命令格式完全兼容

### 建议

**方案 A：符号链接共享（推荐）**

```bash
# 保留 .claude/commands/tg/ 作为源
# .opencode/commands/tg/ 使用符号链接
cd tg-workflow
rm -rf .opencode/commands
ln -s ../.claude/commands .opencode/commands
```

**优点**：
- 只需维护一份
- 两工具自动同步
- 简单直接

**缺点**：
- Windows 需特殊处理
- 符号链接在某些环境下可能失效

---

**方案 B：Skills 目录迁移**

根据 Claude Code 官方建议，Commands 和 Skills 已统一：

```bash
# 将 commands 迁移到 skills
mv .claude/commands/tg skills/tg-commands
# 创建 SKILL.md 定义
```

**优点**：
- 符合最新规范
- OpenCode 自动发现 `.claude/skills/`

**缺点**：
- 需要重构现有结构
- 改变用户习惯

## 结论-003：Hooks/Plugins 管理

### 发现

| 工具 | Hooks 定义方式 |
|------|---------------|
| Claude Code | `settings.json` 中的 `hooks` 字段 |
| OpenCode | `.opencode/plugins/*.ts` 独立文件 |

### 建议

**保持分离**：
- `hooks/` 目录：Claude Code 专用 JS 脚本
- `plugins/` 目录：OpenCode 专用 TS 插件

**原因**：两个工具的 Hook 机制完全不同，无法统一。

---

## 结论-004：推荐目录结构

```
tg-workflow/
├── AGENTS.md                    # 跨工具核心规则
├── CLAUDE.md → AGENTS.md        # 符号链接（可选）
│
├── .claude/
│   ├── commands/tg/             # Commands（源）
│   └── skills/                  # Skills（两工具共享）
│       ├── tg-proposal/
│       └── tg-memory/
│
├── .opencode/
│   ├── commands → ../.claude/commands  # 符号链接
│   └── plugins/                 # OpenCode 专用
│
├── hooks/                       # Claude Code 专用
│   ├── session-start.js
│   └── session-end.js
│
└── plugins/                     # OpenCode 专用
    └── tg-memory-plugin.ts
```

**变更**：
1. 删除 `skills/` 目录，移动到 `.claude/skills/`
2. `.opencode/commands/` 改为符号链接
3. 添加 `AGENTS.md` 作为跨工具核心
