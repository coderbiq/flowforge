## 问题重新定义

### 用户核心诉求

1. **源仓库目录清晰**：不使用隐藏目录（`.claude/`），使用清晰目录（`configs/claude/`）
2. **按平台分组**：一个平台的所有配置放在一起，便于查看
3. **安装到项目**：复制到目标项目的隐藏目录（`.claude/`、`.opencode/`）
4. **自用模式**：tg-workflow 自己使用时通过软链接

---

## 结论-001：源目录结构设计

### 推荐结构

```
tg-workflow/                           # 源仓库
├── configs/                           # 清晰目录（非隐藏）
│   ├── claude/                        # Claude Code 平台配置
│   │   ├── commands/                  # 命令
│   │   │   └── tg/                    # /tg:* 命令组
│   │   │       ├── apply.md
│   │   │       ├── archive.md
│   │   │       ├── explore.md
│   │   │       ├── list.md
│   │   │       ├── notes.md
│   │   │       ├── propose.md
│   │   │       └── status.md
│   │   ├── skills/                    # 技能
│   │   │   ├── tg-proposal/
│   │   │   └── tg-memory/
│   │   ├── hooks/                     # 钩子脚本
│   │   │   ├── session-start.js
│   │   │   └── session-end.js
│   │   └── settings.json              # Claude Code 配置
│   │
│   └── opencode/                      # OpenCode 平台配置
│       ├── commands/                  # 命令
│       │   └── tg/
│       │       └── ... (同上)
│       ├── skills/                    # 技能（如需要平台特定）
│       ├── plugins/                   # OpenCode 插件
│       │   └── tg-memory-plugin.js
│       └── settings.json              # OpenCode 配置
│
├── docs/                              # 文档
├── templates/                         # 模板
├── scripts/                           # 安装脚本
│   └── install.sh
├── CLAUDE.md                          # 项目说明（或软链接到 AGENTS.md）
└── AGENTS.md                          # 跨工具核心规则
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 源目录命名 | `configs/` | 清晰、非隐藏、避免与 `.config/` 混淆 |
| 平台分组 | `configs/claude/`、`configs/opencode/` | 一个平台所有配置在一起 |
| 命令目录 | `commands/tg/` | 保持子目录结构，支持命名空间 |
| Skills 位置 | `configs/claude/skills/` | 每个平台独立的 skills |

## 结论-002：安装模式设计

### 模式 1：项目安装（复制）

**适用场景**：将 tg-workflow 安装到其他项目

```bash
# 安装到当前项目
./scripts/install.sh project

# 安装到指定项目
./scripts/install.sh project /path/to/project
```

**安装后目录**：

```
目标项目/
├── .claude/                    # 隐藏目录
│   ├── commands/tg/           # 从 configs/claude/commands/ 复制
│   ├── skills/                # 从 configs/claude/skills/ 复制
│   ├── hooks/                 # 从 configs/claude/hooks/ 复制
│   └── settings.json          # 从 configs/claude/settings.json 复制
└── .opencode/                  # 隐藏目录
    └── ...
```

---

### 模式 2：全局安装（复制）

**适用场景**：用户全局使用 tg-workflow

```bash
./scripts/install.sh global
```

**安装后目录**：

```
~/.claude/                      # 用户全局 Claude Code 配置
├── commands/tg/
├── skills/
└── ...

~/.config/opencode/             # 用户全局 OpenCode 配置
├── commands/tg/
└── ...
```

---

### 模式 3：自用模式（软链接）

**适用场景**：tg-workflow 自身使用这些配置

```bash
# 在 tg-workflow 根目录执行
./scripts/install.sh self
```

**安装后目录**：

```
tg-workflow/
├── configs/                    # 源目录（清晰）
│   ├── claude/
│   └── opencode/
├── .claude → configs/claude    # 软链接
└── .opencode → configs/opencode # 软链接
```

## 结论-003：目录迁移步骤

### 当前 → 目标

| 当前位置 | 目标位置 | 操作 |
|----------|----------|------|
| `.claude/commands/tg/` | `configs/claude/commands/tg/` | 移动 |
| `.opencode/commands/tg/` | `configs/opencode/commands/tg/` | 移动（删除重复） |
| `hooks/` | `configs/claude/hooks/` | 移动 |
| `plugins/` | `configs/opencode/plugins/` | 移动 |
| `skills/tg-proposal/` | `configs/claude/skills/tg-proposal/` | 移动 |
| `skills/tg-memory/` | `configs/claude/skills/tg-memory/` | 移动 |

### 注意事项

1. **删除重复**：`.opencode/commands/tg/` 内容与 `.claude/commands/tg/` 完全相同，只保留一份
2. **Skills 共享**：OpenCode 自动加载 `.claude/skills/`，可以在 `configs/opencode/skills/` 创建软链接指向 `../claude/skills/`
3. **创建软链接**：tg-workflow 自用时创建 `.claude → configs/claude` 软链接

## 结论-004：安装脚本设计

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
CONFIGS_DIR="$SCRIPT_DIR/configs"

# 安装模式
INSTALL_MODE="${1:-help}"
PROJECT_PATH="${2:-$(pwd)}"

# 安装到项目（复制）
install_project() {
  local target="$1"
  echo "Installing to project: $target"
  
  # Claude Code
  mkdir -p "$target/.claude/commands/tg"
  cp -r "$CONFIGS_DIR/claude/commands/tg/"* "$target/.claude/commands/tg/"
  cp -r "$CONFIGS_DIR/claude/skills" "$target/.claude/"
  cp -r "$CONFIGS_DIR/claude/hooks" "$target/.claude/"
  [[ -f "$CONFIGS_DIR/claude/settings.json" ]] && cp "$CONFIGS_DIR/claude/settings.json" "$target/.claude/"
  
  # OpenCode
  mkdir -p "$target/.opencode/commands/tg"
  cp -r "$CONFIGS_DIR/opencode/commands/tg/"* "$target/.opencode/commands/tg/"
  cp -r "$CONFIGS_DIR/opencode/plugins" "$target/.opencode/"
  
  echo "✓ Installed to $target"
}

# 全局安装（复制到用户目录）
install_global() {
  echo "Installing globally to $HOME"
  
  # Claude Code
  mkdir -p "$HOME/.claude/commands/tg"
  cp -r "$CONFIGS_DIR/claude/commands/tg/"* "$HOME/.claude/commands/tg/"
  cp -r "$CONFIGS_DIR/claude/skills" "$HOME/.claude/"
  cp -r "$CONFIGS_DIR/claude/hooks" "$HOME/.claude/"
  
  # OpenCode
  mkdir -p "$HOME/.config/opencode/commands/tg"
  cp -r "$CONFIGS_DIR/opencode/commands/tg/"* "$HOME/.config/opencode/commands/tg/"
  
  echo "✓ Installed globally"
}

# 自用模式（软链接）
install_self() {
  echo "Setting up symlinks for tg-workflow itself"
  
  cd "$SCRIPT_DIR"
  
  # 创建软链接
  [[ ! -L ".claude" ]] && ln -s configs/claude .claude
  [[ ! -L ".opencode" ]] && ln -s configs/opencode .opencode
  
  echo "✓ Created symlinks: .claude → configs/claude, .opencode → configs/opencode"
}

case "$INSTALL_MODE" in
  project) install_project "$PROJECT_PATH" ;;
  global)  install_global ;;
  self)    install_self ;;
  *)
    echo "Usage: $0 {project [path] | global | self}"
    echo ""
    echo "Modes:"
    echo "  project [path]  Install to project (copy to .claude/, .opencode/)"
    echo "  global          Install globally (copy to ~/.claude/, ~/.config/opencode/)"
    echo "  self            Setup for tg-workflow itself (create symlinks)"
    exit 1
    ;;
esac
```

## 已确认决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 目录命名 | `configs/` | 清晰、非隐藏、避免与 `.config/` 混淆 |
| Skills 位置 | 每个平台目录包含完整的 skills | 支持单独安装某个平台 |
| 安装脚本 | 支持单独安装 `claude` 或 `opencode` | 灵活性，不是全量安装 |
| hooks/plugins | 按平台分离 | hooks → `configs/claude/hooks/`，plugins → `configs/opencode/plugins/` |

### 安装脚本设计

```bash
#!/usr/bin/env bash
# 用法：
#   ./install.sh claude [project-path]   # 只安装 Claude Code 配置
#   ./install.sh opencode [project-path] # 只安装 OpenCode 配置
#   ./install.sh all [project-path]      # 安装所有平台配置
#   ./install.sh self                    # tg-workflow 自用（创建软链接）

# 安装 Claude Code 配置
install_claude() {
  local target="${1:-$(pwd)}"
  echo "Installing Claude Code configs to: $target"
  
  mkdir -p "$target/.claude/commands/tg"
  mkdir -p "$target/.claude/skills"
  mkdir -p "$target/.claude/hooks"
  
  cp -r "$CONFIGS_DIR/claude/commands/tg/"* "$target/.claude/commands/tg/"
  cp -r "$CONFIGS_DIR/claude/skills/"* "$target/.claude/skills/"
  cp -r "$CONFIGS_DIR/claude/hooks/"* "$target/.claude/hooks/"
  [[ -f "$CONFIGS_DIR/claude/settings.json" ]] && \
    cp "$CONFIGS_DIR/claude/settings.json" "$target/.claude/"
}

# 安装 OpenCode 配置
install_opencode() {
  local target="${1:-$(pwd)}"
  echo "Installing OpenCode configs to: $target"
  
  mkdir -p "$target/.opencode/commands/tg"
  mkdir -p "$target/.opencode/skills"
  mkdir -p "$target/.opencode/plugins"
  
  cp -r "$CONFIGS_DIR/opencode/commands/tg/"* "$target/.opencode/commands/tg/"
  cp -r "$CONFIGS_DIR/opencode/skills/"* "$target/.opencode/skills/"
  cp -r "$CONFIGS_DIR/opencode/plugins/"* "$target/.opencode/plugins/"
  [[ -f "$CONFIGS_DIR/opencode/settings.json" ]] && \
    cp "$CONFIGS_DIR/opencode/settings.json" "$target/.opencode/"
}
```
