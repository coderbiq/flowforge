# 发现-006：命令实现方案

**状态**: 已确认
**发现时间**: 2026-05-17
**触发场景**: CR26051701 实施过程中

---

## 问题描述

### 现象

当前将所有 `/tg:*` 命令写在一个 `tg-proposal` Skill 文件中：

```yaml
---
name: tg-proposal
description: |
  **命令列表**:
  - `/tg:explore` - 创建探索笔记并主动探索
  - `/tg:propose` - 创建提案和任务 Epic
  ...
---
```

### 问题

- AI agent 只能识别为一个 Skill (`tg-proposal`)
- `/tg:explore`, `/tg:propose` 等命令不会被识别为独立命令
- 用户无法直接调用 `/tg:*` 格式的命令

---

## 解决方案

### 核心机制：目录结构决定命令 ID

**OpenCode**：
- `.opencode/commands/tg/explore.md` → `project:tg:explore`
- 目录分隔符 `/` 替换为冒号 `:`
- 所有命令有 `project:` 或 `user:` 前缀
- 实际调用：`/project:tg:explore`

**Claude Code**：
- `.claude/commands/tg/explore.md` → `/tg:explore`
- 目录分隔符 `/` 替换为冒号 `:`
- 无强制前缀
- 实际调用：`/tg:explore`

---

## 推荐实现

### 目录结构

```
.claude/commands/tg/
├── explore.md     →  /tg:explore
├── propose.md     →  /tg:propose
├── apply.md       →  /tg:apply
├── archive.md     →  /tg:archive
├── status.md      →  /tg:status
├── list.md        →  /tg:list
└── notes.md       →  /tg:notes

.opencode/commands/tg/
├── explore.md     →  project:tg:explore
├── propose.md     →  project:tg:propose
└── ... (同上)
```

### 命令文件结构

```markdown
---
description: 创建探索笔记并主动探索
allowed-tools: Skill(tg-proposal)
---

Use the `tg-proposal` skill to create exploration notes and actively explore.

Arguments: $ARGUMENTS
```

### Skill 角色

- `tg-proposal` Skill 作为实现层
- 命令作为薄包装层，委托给 Skill
- 复杂逻辑在 Skill 中实现

## 当前 description 问题分析

### 问题

当前 tg-proposal description 中命令格式错误：

```yaml
**命令列表**:
- `/tg:explore` - 创建探索笔记并主动探索
- `/tg:propose` - 创建提案和任务 Epic
```

**问题**：
1. `/tg:explore` 格式在 Claude Code 中不存在（应该是 `/tg-explore`）
2. 在 OpenCode 中是 `/project:tg:explore`（有 `project:` 前缀）
3. 用户无法用这些命令触发 Skill

### 正确方案

**命令 + Skill 双层架构**：

1. **命令文件**（`.claude/commands/tg/*.md`）：
   - 作为入口，可被用户直接调用
   - 格式：`/tg:explore`（Claude Code）

2. **Skill 文件**（`skills/tg-proposal/SKILL.md`）：
   - 作为实现层，包含复杂逻辑
   - 命令委托给 Skill 执行

3. **Description 优化**：
   - 不在 description 中列出命令（命令由命令文件定义）
   - focus on 触发关键词和使用场景

## 优化后的 description 结构

```yaml
description: |
  需求管理核心技能，管理从探索到归档的完整生命周期。

  **显式触发** (用户明确提及):
  - "提案"、"proposal"、"变更请求"
  - "探索需求"、"分析需求"
  - "创建提案"、"归档提案"

  **隐式触发** (场景识别):
  - 用户描述新功能需求
  - 用户询问"如何实现..."
  - 用户提到"调研"、"探索"、"分析"
```

**优化要点**：
- 移除命令列表（命令由命令文件定义）
- 添加显式触发关键词
- 简化核心定位
- 保持多行 YAML 格式
