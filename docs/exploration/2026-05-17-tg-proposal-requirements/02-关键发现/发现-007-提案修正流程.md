# 发现-007：提案修正流程

**状态**: 已确认
**发现时间**: 2026-05-17
**触发场景**: CR26051701 实施过程中发现设计缺陷

---

## 问题背景

### 错误操作

在实施 CR26051701 过程中发现命令实现方案有问题，我错误地：
1. 创建了新的探索笔记目录 (`2026-05-17-tg-proposal-commands/`)
2. 考虑直接更新提案文档

### 正确操作

1. 在现有提案的探索笔记中追加新发现
2. 完善探索文档
3. 更新提案设计
4. 继续实施

---

## 根本问题

当前工作流设计中缺少：

1. **修订状态**：从 Active 回到探索阶段的机制
2. **回退规则**：实施中发现缺陷时应如何处理
3. **修订历史**：如何在提案中记录修订

---

## 正确的提案修正流程

```
实施中（Active）
    ↓ 发现设计缺陷
回到探索阶段（不创建新探索笔记）
    ↓ 在现有探索笔记中追加发现
探索阶段（追加发现、讨论、确认方案）
    ↓ 更新提案文档（追加修订历史）
提案更新（修订版）
    ↓ 继续实施
实施中（Active）
```

## 补充设计

### 状态流转扩展

```
Draft ←→ Proposed ←→ Active ←→ Implemented
                      ↓
                 Revised（修订中）
                      ↓
               回到探索阶段
```

### 新增规则

1. 实施中发现缺陷 → 在现有探索笔记中追加发现
2. 不创建新的探索笔记目录
3. 更新提案时追加修订历史章节
4. 如果是复杂问题，需要多次讨论后再更新提案
