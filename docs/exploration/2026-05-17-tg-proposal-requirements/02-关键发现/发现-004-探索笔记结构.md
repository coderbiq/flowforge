# 发现-004：探索笔记结构

**发现时间**: 2026-05-17 15:16

---

## 核心洞察

探索笔记采用"混合模式"结构，解决时间线记录在复杂探索中过长的问题。

---

## 问题

用户担忧：
> "探索日志目前被设计成 timeline 的记录方式对于复杂的探索会不会太长了"

---

## 研究结果

研究了探索笔记/研究笔记的结构最佳实践，发现三种主要模式：

### 模式 1：原子化笔记 + 结构笔记（Zettelkasten）

```
每个发现一个独立笔记
用链接关联
避免长文档
```

### 模式 2：渐进式总结（Progressive Summarization）

```
Layer 0: 原始来源
Layer 1: 关键段落
Layer 2: 粗体高亮（10-20%）
Layer 3: 高亮精髓（2-4%）
Layer 4: 执行摘要
```

### 模式 3：ADR 模式

```
聚焦决策
简短（1-2页）
包含上下文、决策、后果
```

---

## 推荐方案：混合模式

```
探索项目/
├── 00-探索概览.md          # 项目级总结和导航
├── 01-探索会话/            # 按时间组织的原始记录（简短）
├── 02-关键发现/            # 原子化笔记
├── 03-探索结论/            # ADR 风格的结论记录
└── 04-结构笔记/            # 主题入口点（MOC）
```

---

## 核心原则

1. **分离原始记录和提炼结果** - 时间线只存在于会话日志
2. **按需投入** - 用渐进式总结避免过度整理
3. **原子化发现** - 每个洞察独立可检索
4. **结构化导航** - 用结构笔记而非文件夹组织
5. **聚焦决策** - 用 ADR 记录有价值的结论

---

## 为什么重要

- 时间线过长的问题转移到简短的会话日志中
- 关键发现独立可检索，不会被埋没在长文档里
- 支持从探索到决策的完整流程
