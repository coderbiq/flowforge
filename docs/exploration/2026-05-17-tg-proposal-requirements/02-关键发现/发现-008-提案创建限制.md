# 发现-008：提案创建限制（铁律）

**状态**: 已确认
**发现时间**: 2026-05-17
**来源**: 用户反馈

---

## 铁律定义

> **在提案实施过程中，如果要创建新提案，必须经过用户同意。**

---

## 背景

在实施 CR26051701 过程中，我错误地考虑创建新的探索笔记目录（虽然没有实际创建提案文件，但思路是错误的）。

这个铁律的目的是：
1. 避免实施过程中创建不必要的提案
2. 确保提案聚焦于当前工作
3. 防止工作流混乱

---

## 适用场景

| 场景 | 操作 | 是否需要用户同意 |
|------|------|-----------------|
| 实施中发现设计细节错误 | 更新提案设计 | 否 |
| 实施中发现方法调整 | 更新提案设计 | 否 |
| 实施中发现范围扩大 | 评估是否需要新提案 | **是** |
| 实施中发现意图改变 | 创建新提案 | **是** |
| 实施中发现完全不相关的问题 | 创建新提案 | **是** |

---

## 实现方式

### 在 tg-proposal Skill 中添加边界规则

```markdown
### 🚫 绝不执行

- 在实施阶段创建新提案（必须先询问用户）

### ⚠️ 先询问

- 创建新提案（任何阶段）
```

### 在 PROPOSAL-WORKFLOW.md 中添加

```markdown
## 边界规则

### 实施阶段铁律

1. **禁止在实施阶段创建新提案**
   - 发现问题 → 在现有探索笔记中追加发现
   - 更新现有提案 → 继续实施
   - 如果需要创建新提案 → 必须经过用户同意
```
