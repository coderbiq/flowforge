# 发现-001：命令前缀统一

**发现时间**: 2026-05-17 14:35

---

## 核心洞察

所有命令应使用 `tg` 前缀，而非 `propose` 前缀，体现 tg-workflow 工作流的整体性。

---

## 背景

用户在探索过程中提出：
> "我希望所有命令以 tg 为前缀"

原设计使用 `/propose:*` 命令前缀，但这与 tg-workflow 的整体命名不一致。

---

## 决策

命令命名从 `/propose:*` 改为 `/tg:*`：

| 原命令 | 新命令 |
|--------|--------|
| `/propose:explore` | `/tg:explore` |
| `/propose:new` | `/tg:propose` |
| `/propose:apply` | `/tg:apply` |
| `/propose:archive` | `/tg:archive` |
| `/propose:status` | `/tg:status` |
| `/propose:notes` | `/tg:notes` |
| `/propose:list` | `/tg:list` |

---

## 影响范围

需要更新的文档：
- tg-workflow/README.md
- tg-workflow/docs/ARCHITECTURE.md
- tg-workflow/docs/PROPOSAL-WORKFLOW.md
- tg-workflow/docs/GETTING-STARTED.md

---

## 为什么重要

统一命名体现工作流的整体性，让用户清楚这是 tg-workflow 的完整方法论，而非孤立的功能模块。
