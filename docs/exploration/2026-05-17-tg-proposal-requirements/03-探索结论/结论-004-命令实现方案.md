# 结论-004：命令实现方案

**状态**: 已确认
**决策时间**: 2026-05-17

---

## 决策

使用目录结构实现 `/tg:*` 命令前缀。

---

## 实现方式

### Claude Code

```
.claude/commands/tg/
├── explore.md     →  /tg:explore
├── propose.md     →  /tg:propose
├── apply.md       →  /tg:apply
├── archive.md     →  /tg:archive
├── status.md      →  /tg:status
├── list.md        →  /tg:list
└── notes.md       →  /tg:notes
```

**调用格式**: `/tg:explore`, `/tg:propose` 等（无强制前缀）

### OpenCode

```
.opencode/commands/tg/
├── explore.md     →  project:tg:explore
├── propose.md     →  project:tg:propose
└── ... (同上)
```

**调用格式**: `/project:tg:explore`, `/project:tg:propose` 等（有 `project:` 前缀）

---

## 核心机制

**目录分隔符替换为冒号**：
- `commands/tg/explore.md` → `/tg:explore` (Claude Code)
- `commands/tg/explore.md` → `project:tg:explore` (OpenCode)

---

## 命令文件结构

```markdown
---
description: 创建探索笔记并主动探索
allowed-tools: Skill(tg-proposal)
---

Use the `tg-proposal` skill to create exploration notes.

Arguments: $ARGUMENTS
```

---

## Skill 角色

- `tg-proposal` Skill 作为实现层（保持单文件）
- 命令文件作为薄包装层，委托给 Skill
- 复杂逻辑在 Skill 中实现

---

## 多平台兼容性

| 平台 | 命令位置 | 调用格式 |
|------|---------|---------|
| Claude Code | `.claude/commands/tg/*.md` | `/tg:*` |
| OpenCode | `.opencode/commands/tg/*.md` | `/project:tg:*` |
