# 结论-001：命令体系

**状态**: 已确认
**决策时间**: 2026-05-17

---

## 决策

tg-proposal Skill 采用以下命令体系：

| 命令 | 阶段 | 说明 |
|------|------|------|
| `/tg:explore` | 探索 | 创建探索笔记，主动探索 |
| `/tg:propose` | 提案 | 创建提案 + 任务 Epic |
| `/tg:apply` | 实施 | 拆解任务，开始实施 |
| `/tg:archive` | 归档 | 归档提案，更新模块文档 |
| `/tg:status` | 状态 | 查看提案状态 |
| `/tg:list` | 列表 | 列出所有提案 |
| `/tg:notes` | 笔记 | 添加实施笔记 |

---

## 理由

1. **统一前缀**：`tg` 前缀体现 tg-workflow 工作流的整体性
2. **简洁明了**：命令名直接表达意图
3. **可扩展**：后续可添加 `/tg:*` 命令

---

## 影响范围

需要更新 tg-workflow 下的所有文档中的命令示例。
