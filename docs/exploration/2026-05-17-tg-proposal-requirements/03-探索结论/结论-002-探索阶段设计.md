# 结论-002：探索阶段设计

**状态**: 已确认
**决策时间**: 2026-05-17

---

## 决策

探索阶段采用"立场而非工作流"理念，AI 主动探索。

---

## 核心立场

1. **Curious, not prescriptive** - 自然地提问，不按剧本
2. **Open threads, not interrogations** - 呈现多个方向，让用户选择
3. **Visual** - 大量使用图表澄清思考
4. **Adaptive** - 跟随有趣的线索，新信息出现时转向
5. **Patient** - 不急于得出结论，让问题形态自然浮现
6. **Grounded** - 探索实际代码，而不只是理论

---

## 行为边界

| ✅ 允许 | ❌ 禁止 |
|--------|--------|
| 读取文件 | 编写代码 |
| 搜索代码 | 实现功能 |
| 调研网络 | 自动保存工件 |
| 映射架构 | 假装理解 |
| 可视化思考 | 强制结构 |
| 提出问题 | 急于结论 |

---

## 核心原则

- 探索阶段只记录不执行
- 变更范围需完整记录到探索笔记
- 待执行变更在创建提案时统一处理

---

## 探索笔记结构

采用混合模式：
- 00-探索概览.md（总结）
- 01-探索会话/（时间线）
- 02-关键发现/（原子化笔记）
- 03-探索结论/（ADR 风格）
- 04-结构笔记/（MOC 入口）
