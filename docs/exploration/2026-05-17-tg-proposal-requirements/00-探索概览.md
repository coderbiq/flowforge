### 发现-006：命令实现方案问题

**现象**：
- 当前将所有 `/tg:*` 命令写在一个 `tg-proposal` Skill 中
- AI agent 只能识别为一个 Skill，无法识别为多个独立命令

**问题**：
- `/tg:explore`, `/tg:propose` 等命令不会被识别为独立命令
- 用户无法直接调用这些命令

**待探索**：
- OpenCode/Claude Code 的命令注册机制
- 多命令 Skill 的正确实现方式
- 命令别名/前缀的实现方案

### 发现-007：提案修正流程

**现象**：
- 实施过程中发现设计缺陷时，我错误地创建了新的探索笔记目录
- 正确做法应是在现有提案的探索笔记中追加新发现

**根本问题**：
- 当前工作流设计中缺少"实施过程中修正提案"的规则
- 缺少"修订"状态和回退机制

**正确流程**：
```
实施中发现问题 → 回到探索阶段 → 追加发现到现有探索笔记 → 更新提案 → 继续实施
```

### 发现-008：提案创建限制（铁律）

**规则**：
> 在提案实施过程中，如果要创建新提案，必须经过用户同意。

**原因**：
- 避免实施过程中创建不必要的提案
- 确保提案聚焦于当前工作
- 防止工作流混乱
