# Extraction Guide

## Knowledge Unit Criteria

Each extracted unit must be:
- **Self-contained**: understandable without reading the source
- **Singular**: one assertion per unit
- **Addressable**: can be independently referenced and linked

Restate in your own words. Do not copy-paste from the source.

## Card Type Mapping

| If the unit is... | Use card type |
|---|---|
| Constraint on future work | `convention` |
| Accepted technical/product decision | `decision` |
| Reusable fact or caveat | `finding` |
| Guiding principle | `convention` |
| Reusable design pattern | `design` |
| Module/system knowledge | `module` |

## Source Tracking

Record the source location (section/paragraph) for each unit. Use `--source-card` to link each created card to the plan card.
