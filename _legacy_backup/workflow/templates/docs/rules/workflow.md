# Global Workflow Rules

## Ownership summary

- Primary module: none
- System / architecture targets: none
- Convention targets: none
- Canonical reading path: this global workflow rule file

## Defaults

- Prefer concise, decision-oriented writing over transcript dumps.
- Keep durable statements attributable to a finding, decision, or explicit
  assumption.
- Use relative links between artifacts so references remain navigable in Git.
- Treat the archived knowledge base as the starting baseline for new work.
- Treat explorations and proposals as delta records against the canonical
  corpus.
- Keep implementation logs out of exploration and proposal bodies.

## Rule boundary

- Core workflow files define the required lifecycle, schema, and validation
  mechanics.
- This file defines the project's default working posture on top of that core.
- Project-local rule edits may refine emphasis and writing style, but they must
  not violate the core lifecycle or metadata contracts.
