## 社区探索 v2：同类高星项目调研（已修正对标方向）
- v1 方向错误：误将 FlowForge 对标为 SKILL 机制项目（anthropics/skills 等）
- v2 修正：FlowForge 是做 SKILL 里的**内容**——项目开发过程管理（分析、设计、执行管理、知识沉淀）
- 正确对标项目：
  - **Superpowers** (219k ⭐) — 最接近，技能触发式工作流（brainstorming→plans→execute→TDD→review），但知识沉淀弱
  - **OpenSpec** (53.1k ⭐) — 知识沉淀最强，proposal + design + tasks + specs + archive 完整链路
  - **Spec-Kit** (109k ⭐) — 最企业化，7 阶段门控式 SDD，但较重
  - **BMad Method** (~4k ⭐) — 21 Agent 企业框架
  - **GSD** (11.9k ⭐) — 元提示波次式，独立开发者友好
  - **Kilo Code** (~8k ⭐) — Memory Bank 知识保留
- 核心发现：FlowForge 是唯一将「CLI 任务管理 + 非线性工作流 + 结构化知识沉淀 + 可部署工具包」整合在一起的项目
- 关键差异化：feedback SKILL 提供迭代闭环（其他项目都是线性/单向流程），library/wiki 提供持续知识积累
